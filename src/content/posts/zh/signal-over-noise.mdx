## 为什么很多博客看起来都“能用，但不想读”

问题往往不在于内容，而在于界面没有帮助内容被阅读。视觉焦点太散、层级不稳定、留白不够，都会让文章失去节奏。

我更喜欢把博客当成一件产品去设计。产品不是越复杂越高级，而是越能让用户在第一眼就知道什么值得看、该从哪里进入。

## 我对个人博客的一个判断

如果一个网站同时承担“作品展示”“知识沉淀”“个人名片”三种角色，那它的界面就必须既有编辑感，也有产品感。

- 编辑感保证阅读舒适度。
- 产品感保证导航和切换足够清晰。
- 品牌感保证访客能记住你是谁。

## 为什么我喜欢静态站点

静态博客的美感，不止是加载快。更重要的是它天然适合长期维护，你可以把内容视为版本化资产，而不是一套越来越重的后台系统。

```ts
const principles = {
  calm: true,
  clear: true,
  maintainable: true
};
```

## 结语

真正耐看的博客，往往不会用很多炫技元素去证明自己。它只是安静地把每一层信息都摆在了正确的位置上。
